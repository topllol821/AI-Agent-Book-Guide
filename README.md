## 這本書在說什麼？

AI Agent 不是一個突然出現的新名詞。它是一步一步演進來的。

- LLM 能回答問題，但記不住你。→ 所以有了 Context Engineering。
- Context 能給資訊，但每次對話都忘。→ 所以有了 Memory 和 Knowledge Base。
- 有記憶但無法做事。→ 所以有了 Tools。
- 有工具但寫程式還是太慢。→ 所以有了 Coding Agent。
- Coding Agent 很強，但你怎麼知道它做對了？→ 所以有了 Evaluation。
- 評估知道好壞，但一個 Agent 能力有限。→ 所以有了 Multi-Agent。
- 多 Agent 能分工，但還是需要人改 Prompt。→ 所以有了 Self Improvement。
- 以上都在文字世界，但真實世界有圖片和語音。→ 所以有了 Multimodal。

**每一項技術，都是為了解決上一代 Agent 的限制而誕生。**

這本導讀要介紹的不是 AI 名詞，而是：**AI Agent 如何一步一步演進成真正能工作的系統。**

如果你是：

- 已經會用 ChatGPT，想知道「Agent 跟 ChatGPT 到底差在哪？」
- 在 GitHub 上看過 Agent 專案，名詞都認識，但拼不起來
- 沒有 AI 工程背景，但希望建立完整的 Agent 觀念
- 不想讀大量英文或太學術的內容

這本就是為你寫的。

## 閱讀方式

### 路徑一：從頭讀（推薦）

如果你是第一次接觸 Agent，建議照章節順序閱讀：

```
01 → 02 → 03 → 04 → 05 → 06 → 07 → 08 → 09 → 10
```

### 路徑二：挑著讀

如果你對某個主題特別感興趣：

- **只想懂核心概念**：01 → 02（先讀這兩章就夠了）
- **想知道 Agent 能做什麼**：04 → 05
- **想了解多 Agent 協作**：07
- **想了解最新發展**：08 → 09
- **想知道 Agent 怎麼變強**：08
- **想略懂模型訓練（延伸閱讀）**：10

### 路徑三：當參考書

每章最後都有「學完本章後應做到」，你可以快速確認自己是否掌握了該章重點。

### 每章閱讀時間

- 01-04：約 15-20 分鐘（核心觀念）
- 05-06：約 20-30 分鐘（進階主題）
- 07-09：約 10-15 分鐘（協作、多模態、進化）
- 10：約 5 分鐘（延伸閱讀）

## 開發方式

本 Repository 採用 **Review-Driven Development** 流程：

```
OpenCode 撰寫 → Review → 修正 → 完成
```

每次撰寫數章後，經 Review 確認方向與品質，再繼續下一階段。避免一次完成整本導致方向偏差或品質不均。

Review 記錄存放在 [`review/`](review/) 目錄中。
