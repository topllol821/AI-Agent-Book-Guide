# AI Agent Book 導讀

> 為中文讀者重新組織的 AI Agent 學習指南
>
> 基於李博杰《深入理解 AI Agent：設計原理與工程實踐》重新編排

---

## 為什麼需要這本導讀？

原書《深入理解 AI Agent》是一本內容紮實的技術書，但它的章節安排是「理論→拆解→進階」，目標讀者是已有 AI 工程背景的開發者。

但我們發現，更多想了解 AI Agent 的人其實是：

- **已經會用 ChatGPT**，想知道「Agent 跟 ChatGPT 到底差在哪？」
- **在 GitHub 上看過 Agent 專案**，名詞都認識，但拼不起來
- **沒有 AI 工程背景**，但希望建立完整的 Agent 觀念
- **不想讀太學術或大量英文的內容**

這本導讀就是為了這些人寫的。

## 本導讀 vs 原書

| | 原書 | 本導讀 |
|---|---|---|
| **讀者定位** | AI 工程師、研究員 | ChatGPT 使用者、開發者新手 |
| **章節順序** | 理論→拆解→進階 | 最易懂→最深入 |
| **寫作風格** | 技術嚴謹、中英夾雜 | 全中文、多案例、多表格 |
| **補充內容** | 作者個人觀點 | 融合 OpenAI / Anthropic / Google 當前共識 |
| **過時處理** | 出版時的觀點 | 若已過時會明確補充說明 |
| **每章結構** | 自由形式 | 固定 10 段格式 |

## 適合哪些人閱讀？

- ✅ 你已經用過 ChatGPT / Claude / Gemini
- ✅ 你想知道 Agent、MCP、RAG 這些名詞到底在說什麼
- ✅ 你想看懂 Cursor、Manus、Deep Research 背後的原理
- ✅ 你不想直接啃原文技術書
- ❌ 你已經是 AI Agent 工程師——建議直接讀原書

## 閱讀方式

### 路徑一：從頭讀（推薦）

如果你是第一次接觸 Agent，建議照章節順序閱讀：

```
01 → 02 → 03 → 04 → 05 → 06 → 07 → 08 → 09 → 10
```

### 路徑二：挑著讀

如果你對某個主題特別感興趣：

- **只想懂核心概念**：01 → 02（先讀這兩章就夠了）
- **想知道 Agent 能做什麼**：04 → 05
- **想了解多 Agent 協作**：07
- **想了解最新發展**：07 → 08
- **想知道 Agent 怎麼變強**：09
- **想略懂模型訓練（延伸閱讀）**：10

### 路徑三：當參考書

每章最後都有「學完本章後應做到」，你可以快速確認自己是否掌握了該章重點。

### 每章閱讀時間

- 01-04：約 15-20 分鐘（核心觀念）
- 05-06：約 20-30 分鐘（進階主題）
- 07-09：約 10-15 分鐘（協作、多模態、進化）
- 10：約 5 分鐘（延伸閱讀）

## 開發方式

本 Repository 採用 **Review-Driven Development** 流程：

```
OpenCode 撰寫 → Review → 修正 → 完成
```

每次撰寫數章後，經 Review 確認方向與品質，再繼續下一階段。避免一次完成整本導致方向偏差或品質不均。

Review 記錄存放在 [`review/`](review/) 目錄中。
