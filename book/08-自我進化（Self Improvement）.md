# 08 — 自我進化（Self Improvement）——越用越聰明

> 本章對應原書第九章（自我進化）。核心問題是：**Agent 如何不靠人類介入，自己變得更強？**

## 這章主要回答什麼問題？

為什麼不能一直靠人來改 Prompt？

Agent 可以自己發現自己錯了，然後自己修正嗎？

自我進化目前有哪些做法？有哪些限制？

## 英文名詞（附中文）

| 英文 | 中文 | 一句話解釋 |
|------|------|-----------|
| Self-Evolution | 自我進化 | Agent 從自己的經驗中學習，不需要人類手動調整 |
| Externalized Learning | 外部化學習 | 把學到的東西存成檔案、知識庫、工具，不改模型權重 |

| Active Tool Discovery | 主動工具發現 | Agent 發現能力不足時，自己去找新工具 |

## 作者真正想表達什麼？

### 概念誕生故事：為什麼不能一直靠人改 Prompt？

**第一階段：人工調 Prompt**

```
一個 Agent 上線了。

第一天：使用者回報「它答錯了」。
你改了 Prompt。

第二天：使用者又回報「它在另一個情境答錯了」。
你又改了 Prompt。

一個月後：
  你的 Prompt 從 50 行變成 500 行。
  你已經搞不清楚每一行在幹嘛。
  改了一處，另一處壞掉。

問題：
  ① 人工調 Prompt 的速度跟不上使用者發現問題的速度
  ② Prompt 越長越難維護
  ③ 你在同一個 Prompt 裡塞了太多 rules，互相衝突
```

**第二階段：規則寫進程式碼**

```
你把常見的錯誤規則寫成程式碼邏輯：
  if 使用者說 A 情況 → 這樣回應
  if 使用者說 B 情況 → 那樣回應

問題：
  ① 你永遠無法預測所有邊界情況
  ② 規則互相衝突時，程式碼邏輯比 Prompt 更難除錯
  ③ LLM 的隨機性讓規則無法完全覆蓋
```

**第三階段：讓 Agent 自己學習**

```
你發現：
  → 人無法預測所有情況
  → 規則是死的，世界是活的
  → 每一筆使用者互動都是學習材料

所以你需要的是：
  讓 Agent 從每一次的成功和失敗中，自己提煉出規則。
  不是人寫規則給 Agent，而是 Agent 自己「長出」規則。
```

這就是自我進化（Self-Evolution）的起點。

### 但是——學習不會自動發生

這是最重要的一句話。

你可能會想：如果 Context 可以無限長，讓 Agent 看過所有歷史不就好了嗎？

**不行。** 注意力機制擅長「查找」（在 1 萬行對話中找到那一行），但不擅長「歸納統計」（從 1000 次經驗中總結規律）。

```
資訊就在那裡，但不提煉成可複用的規律，就是沒用。
```

### 重要的安全提醒

自我改善是進階能力。**未經評估與授權的自動修改，可能讓系統退化、偏離原本規則或引入安全風險。**

因此，任何自我進化機制都應維持在受控閉環中運作：

```text
蒐集失敗案例
↓
產生候選改善方案
↓
離線評估與安全檢查
↓
人類核准或受控發布
↓
持續監控是否退化
```

### 三種學習範式

| 範式 | 改什麼 | 優點 | 缺點 |
|------|--------|------|------|
| 後訓練（Post-training） | 模型權重 | 效果最持久 | 成本高、週期長、需要 GPU |
| 上下文學習（In-Context Learning） | 當前的 Prompt | 見效快、零成本 | 對話結束就消失 |
| **外部化學習（Externalized Learning）** | **模型之外的檔案、知識庫、工具** | **持久、可解釋、可修正** | **需要設計學習機制** |

本章聚焦的是第三種：**不改模型權重，讓 Agent 透過外部化學習變強。**

## 白話解釋

### 外部化學習就像「寫工作日記」

想像一個新的客服 Agent 第一天上班：

```
第一次處理退款：
  花了 15 分鐘——查了 3 次政策、打了 2 通電話給主管。
  終於成功退款。

如果它沒有學習能力：
  第二次遇到一樣的情況——又花 15 分鐘從頭來。
  這次累積的經驗，隨著對話結束就消失了。

如果有外部化學習：
  它會把學到的東西寫進「工作日記」：
  → 「銀行 B 的退款需要先驗證信用卡後四碼」
  → 下次遇到銀行 B，直接知道第一步做什麼
  → 處理時間從 15 分鐘降到 1 分鐘
```

### Reflection：讓 Agent 自己回顧

Reflection 是最基礎的自我進化方式。Agent 完成任務後，會自己回顧一遍：

```
任務：幫使用者查機票

執行過程：
  1. 使用者說「下週去東京」
  2. Agent 查了成田機場的航班
  3. 使用者說「我是指羽田」

Reflection（自我回顧）：
  ✓ 我查到了航班
  ✗ 但我沒有先確認使用者要去哪個機場
  → 下次應該先確認機場，再查航班
```

這個反思結果可以存進 Memory 或 Strategy Summary，下次類似的任務就會用上。

### 三種經驗沉澱方式

| 沉澱方式 | 適合的經驗類型 | 範例 |
|---------|--------------|------|
| 知識庫條目 | 純事實和規則 | 「銀行 B 的退款需要信用卡後四碼」 |
| 專用工具（程式碼） | 經常使用的操作流程 | 「查詢帳戶餘額的 API 呼叫序列」 |
| Skill 文件 | 複雜但常變的策略 | 「處理跨國退款的最佳流程」 |

判斷該用哪種的經驗法則：純事實存知識庫，經常用且參數複雜的寫成程式碼（工具），經常變且涉及策略判斷的寫成文件（Skill）。

## 真實案例

### 案例一：從失敗中學習（生活）

你用 Agent 訂餐廳。Agent 訂錯了日期——你訂的是「下週五」，但 Agent 訂成了「這週五」。

有自我進化的 Agent 會：

1. 你糾正它：「我是說下週五。」
2. Agent 反思：「我理解錯了『下週』的意思。下次使用者說『下週 X』，要確認具體日期。」
3. 這條規則寫進 Memory。

下次你說「幫我訂下下週二的餐廳」，它會先問你：「下下週二是 8 月 5 日，對嗎？」

### 案例二：系統提示詞自動優化（示意）

想像一個客服 Agent 遇到政策爭議時，總是直接轉接給人類客服。

人類專家反饋後，Coding Agent 可以做以下事情：

1. 讀取目前的 System Prompt
2. 找到「遇到爭議時轉接人類客服」的相關規則
3. 修改規則：「除非使用者明確要求轉接，否則先嘗試用政策文件回答」
4. 跑評估測試驗證修改效果
5. 確認沒有 Regression 後才更新 Prompt

形成「反饋 → 改寫 → 驗證」的閉環。

### 案例三：System Prompt 自動優化（示意情境）

一家公司發現 Agent 頻繁犯同一類錯誤。

傳統做法：工程師手動改 System Prompt，手動測試，手動部署。

自我進化的受控做法：

1. **收集失敗案例**：系統自動標記失敗的對話
2. **分析錯誤模式**：LLM 分析失敗案例，歸納出「錯誤類型」
3. **產生修正方案**：Coding Agent 針對錯誤類型修改 System Prompt
4. **離線評估**：用歷史案例驗證修正效果，檢查是否退化
5. **人類核准或受控發布**：經確認後才部署
6. **持續監控**：觀察是否發生新的退化或副作用

### 案例四：睡眠整合（AI）

一個客服 Agent 每天處理上萬次對話。

白天：不斷累積新的經驗和記憶。

晚上（低峰期）：

1. 系統總結當天的新經驗
2. 歸納出常見的錯誤模式
3. 壓縮重複的記憶
4. 建立新的策略摘要
5. 為第二天的對話準備好更精簡的 Context

這就是「睡眠整合」——人類需要睡眠來鞏固記憶，Agent 也需要。

## 常見誤解

### ❌ 迷思一：Agent 會自動從經驗中學習

不對。學習必須被顯式設計出來。把原始經驗堆進 Context，模型能「記得」卻不會自動「提煉」。

### ❌ 迷思二：自我進化 = 重新訓練模型

不對。本章討論的是不改權重的外部化學習——把知識和流程存到模型之外。重新訓練模型是第十章（延伸閱讀）的範圍。

### ❌ 迷思三：記憶就是學習

不對。第三章的記憶是「記得更多資訊」，本章的自我進化是「變得更能幹」。兩者層次不同。

### ❌ 迷思四：System Prompt 自動優化 = 亂改自己的設定

不對。即使是自動優化也應維持受控閉環——蒐集案例、離線評估、人類核准、持續監控。不是讓 Agent 在生產環境中未經授權修改自己的行為規則。

## 一句話記住

> **學習不會自動發生，必須被顯式設計。不改變模型權重，而是把經驗沉澱成知識庫、工具和 Skill 文件，讓 Agent 從聰明變成熟練。**

## 本章重點

1. **學習不會自動發生，必須被顯式設計**——這是最重要的認知

2. **三種範式**：後訓練（改權重、成本高）、上下文學習（臨時、見效快）、外部化學習（持久、可解釋）

3. **外部化學習的三種產物**：知識庫（事實）、工具（流程）、Skill（策略）

4. **Reflection（反思）**：Agent 回顧自己的執行過程，找出可以改進的地方

5. **從成功和失敗中都能學**：成功提煉策略摘要，失敗建立錯誤模式庫

6. **System Prompt 自動優化**：形成「反饋 → 改寫 → 驗證」的閉環

7. **睡眠整合**：在背景定期整理和歸納積累的經驗

8. **主動工具發現**：Agent 意識到能力缺口時自己去找新工具

9. **外部化學習讓 Agent 從「聰明」變成「熟練」**

10. **人類只在最終確認時介入**，不用手動改每一條規則
