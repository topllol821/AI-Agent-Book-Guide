# 10 — 模型後訓練（延伸閱讀）——了解模型如何變強

## 這章主要回答什麼問題？

為什麼模型需要後訓練？

SFT、RLHF、Constitutional AI、Preference Optimization 各自在解決什麼問題？

這些東西跟 Agent 有什麼關係？

## 英文名詞（附中文）

| 英文 | 中文 | 一句話解釋 |
|------|------|-----------|
| SFT (Supervised Fine-Tuning) | 監督微調 | 給模型標準答案範例，讓它學會回答格式 |
| RLHF (Reinforcement Learning from Human Feedback) | 人類反饋強化學習 | 讓人類對模型的回答打分，模型學會偏好 |
| Constitutional AI | 憲法 AI | 給模型一套規則，讓它自己判斷什麼是對的 |
| Preference Optimization | 偏好優化 | 比 RLHF 更簡單的訓練方式（如 DPO） |
| Post-training | 後訓練 | 預訓練之後的所有訓練步驟的總稱 |

## 為什麼這章是延伸閱讀？

**對於使用 AI Agent 的一般人，你不需要學會怎麼訓練模型。**

這章的存在是為了讓你知道：**你用的模型不是從石頭裡蹦出來的。**

它經過了複雜的訓練流程。知道這些，能幫助你理解模型的限制在哪、什麼時候該期待它進步。

## 一句話記住這句話

> **SFT 使用高品質示例塑造任務行為；RL 與偏好優化透過比較調整偏好。兩者都可能泛化，也都可能過度擬合。**

SFT 像老師給標準答案讓學生學習——見效快，但可能只學會表面形式。偏好優化讓模型在不同回答間學會選擇——但偏好不代表真理。

## 三階段訓練流水線

```
第一階段：預訓練（Pre-training）
  模型在海量網際網路文字上學會語言和知識。
  → 成本最高（數千萬美元算力）
  → 產出：一個什麼都知道但不太聽話的模型

第二階段：SFT（監督微調）
  用幾千到幾萬條「問題-標準答案」範例。
  → 教會模型回答問題的格式
  → 成本：幾小時到幾天
  → 產出：一個會回答問題但不一定準確的模型

第三階段：RL / 偏好優化
  讓模型自己產生多個回答，讓評審（人類或 AI）選出最好的。
  → 教會模型什麼是好回答
  → 成本：SFT 的數十倍
  → 產出：最終的產品級模型
```

### SFT 在解決什麼問題？

預訓練後的模型會「接續文字」，但不一定會「回答問題」。

```
預訓練模型的行為：
  使用者：「台灣的首都是？」
  模型：「台灣的首都是台北，這是一個常見的地理知識問題。」

看起來好像對？但這是模型在「接續文字」，不是「回答問題」。

SFT 要解決的就是這個：教模型知道「當人類問問題時，你要給答案，不是接續寫文章。」
```

### RLHF 在解決什麼問題？

SFT 後的模型會回答問題，但不知道「什麼是好答案」。

```
SFT 模型的問題：
  使用者：「給我一個減肥建議。」
  模型 A：「少吃多運動。」（正確但太短）
  模型 B：「建議你每天運動 30 分鐘，控制飲食，多吃蔬菜水果，
            避免高糖高油食物。如果你需要更詳細的計畫，
            建議諮詢營養師。」（好回答）

人類偏好：模型 B 更好。但 SFT 沒有辦法教這個——因為 SFT 只有「對的答案」，沒有「好的答案」。

RLHF 的做法：
  ① 模型產生多個回答
  ② 人類排序（最喜歡到最不喜歡）
  ③ 模型調整權重，讓它更傾向人類喜歡的回答
```

### Constitutional AI 在解決什麼問題？

RLHF 需要人類大量標註——成本高、速度慢、一致性差。

Constitutional AI 的做法：以原則或規範協助產生、批評、修正與篩選訓練資料或模型行為。

```
憲法範例：
  「你的回答應該是有幫助的、誠實的、無害的。」

流程概念：
  ① 模型產生回答
  ② 根據憲法原則對回答進行批評和修正
  ③ 用修正後的回答作為訓練資料
  ④ 重複上述步驟迭代優化

優點：減少人類大量標註的需求
缺點：憲法的品質直接決定訓練效果
```

### Preference Optimization 在解決什麼問題？

RLHF 的訓練過程複雜、不穩定。Preference Optimization（如 DPO）簡化了這個流程。

```
RLHF 的流程：
   蒐集人類偏好資料 → 訓練獎勵模型 → 用獎勵模型訓練原模型
   （需要額外的獎勵模型，訓練不穩定）

DPO 的流程：
   直接利用偏好資料中的「較佳回答 vs 較差回答」對來優化模型
   （不需要額外訓練獎勵模型，流程更簡化）
```

## 這些跟 Agent 的關係是什麼？

```
作為 Agent 的使用者：
  你知道模型經過了 SFT + RL 的訓練。
  → 這解釋了為什麼模型「通常」會按照你的指示回答
  → 這解釋了為什麼模型有時候會拒絕回答（Constitutional AI 的限制）
  → 這解釋了為什麼模型偏好某些回答風格（RLHF 的人類偏好）

作為 Agent 的開發者：
  你可以透過 SFT 微調模型，讓它在你的特定任務上表現更好。
  → 但這需要技術能力和資料
  → 對多數人來說，用 Prompt + RAG + Tools 的組合比微調更實際
```

## 常見誤解

### ❌ 迷思一：SFT 可以解決所有問題

SFT 用高品質示例塑造模型行為，但如果訓練資料不夠多樣或只涵蓋特定情境，模型對沒見過的問題可能表現不佳。

### ❌ 迷思二：RL 訓練後的模型就是完美的

RL 讓模型學會「人類偏好」，但人類偏好不一定是「真相」。模型可能會變得更會說謊（如果說謊得到高分）、更會討好使用者（而不是更正確）、更保守（避免犯錯所以什麼都不做）。

### ❌ 迷思三：微調比 Context Engineering 重要

對多數應用來說，Context Engineering 的投資報酬率遠高於微調。先把 Context、RAG、Tools 做好，再考慮微調。

## 本章重點

1. **三階段流水線**：預訓練 → SFT → RL / 偏好優化

2. **SFT 用示例塑造行為，偏好優化調整偏好**——兩者都可能泛化或過擬合

3. **SFT**：給模型標準答案，讓它學會回答格式。成本低、見效快

4. **RLHF**：讓人類對回答排序，模型學會偏好。成本高、效果好

5. **Constitutional AI**：給模型規則，讓它自己審查自己。減少人類標註

6. **Preference Optimization（DPO）**：簡化 RLHF，更穩定

7. **對多數人來說，Context Engineering 比微調更重要**

8. **後訓練不能解決所有問題**——它也有偏見和限制

9. **後訓練解釋了為什麼模型「通常」聽話但有時候拒絕回答**

10. **這章只是延伸閱讀——不知道也沒關係**
